

# 客戶管理模組

此模組目的要用於管理服務單的用戶，允許有管理權限的管理者編輯客戶(customers)的資料

## 客戶資料編輯

允許管理員對客戶表單(customers)資料進行新增、修改、刪除(軟刪除)，每個項目都要有對應權限才允許進行操作
- customer.create (已經存在)
- customer.read (已經存在)
- customer.delete(尚未建立)
- customer.update(尚未建立)
### 列表
客戶管理功能的入口，需要有 `customer.read` ，預設會有列表呈現用戶的資料，包含姓名、電話、身分證字號(列表需要部份隱碼顯示前後4碼，後六碼都隱碼)、LINE ID、信箱，並且可以有匯出畫面上顯示的列表excel；必須存在搜尋功能，可以參考服務表單的搜尋方式

### 新增
允許客戶資料被新增，需要有 `customer.create` 才能被操作，功能入口位於列表右上區塊

### 更新
允許客戶資料被修改，需要有 `customer.update` 才能被操作，點選列表上的按鈕進行detail的顯示並且可以編輯

### 刪除
允許客戶資料被刪除，需要有 `customer.delete` 才能被操作，點選列表上的按鈕，跳出警示視窗與提示後，同意後方可以進行軟刪除

# 服務表單客戶模組擴充功能

此模組是為了方便讓系統管理員新增客戶時可以使用AI來協助辨識資料，因此上傳照片時可以上傳正反面身分證，送到後端進行辨識

### 辨識技術採用
- Azure OCR  - 增加一個 config 開關可以決定是否使用(預設關閉)
- Gemini AI 辨識 - 可以提供1~2張圖片進行身分證反面的辨識

辨識完成後不需要再進行信心計算(請移除這部分功能)，由用戶自行辨識確認，僅需要求輸出格式(json)：
```json
{
	"name":string|null,
	"idNumber":string|null,
	"address":string|null
}
```
以上資料會讓前端自動帶入服務單快速建立客戶資料的區塊
